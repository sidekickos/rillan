# ADR-001: OpenAI-Compatible Upstream Is the First Runtime Provider

## Status

Accepted

## Context

Milestone one needs one real outbound provider path, but it should avoid a broad provider matrix. The local Rillan planning docs already require Anthropic to remain non-default and discouraged for now, while the inbound API is OpenAI-compatible `/v1/chat/completions`.

## Decision

Rillan implements a real outbound OpenAI-compatible upstream path first.

Anthropic remains:

- represented in config as an explicit future seam
- never default
- opt-in only at the configuration layer
- not implemented as a milestone-one runtime adapter

## Consequences

- Milestone one stays small because the outbound request shape matches the inbound API closely.
- The HTTP handler can stay provider-agnostic while avoiding premature schema-translation work.
- Anthropic policy is encoded without spending milestone-one effort on a discouraged path.
- Future providers can be added behind the same provider interface when they are worth the maintenance cost.
